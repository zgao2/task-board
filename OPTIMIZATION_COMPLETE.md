### 3. 状态自动更新逻辑

**Store 实现** (`src/store/useStore.ts`):

#### 子任务完成时自动更新:
```typescript
updateSubTaskStatus(taskId, subTaskId, 'done')
→ 自动更新父任务进度
→ 自动释放员工工作量
→ 自动检查是否所有子任务完成
→ 自动完成主任务
→ 自动解锁依赖任务
```

#### 分配任务时自动检查:
```typescript
assignTaskToAgent(taskId, agentId, workload)
→ 检查员工工作量是否超限
→ 自动更新员工负载
→ 自动更新员工状态（在线/忙碌）
→ 创建任务分配记录
```

#### 进度自动计算:
```typescript
progress = (已完成子任务数 / 总子任务数) × 100%
```

## 🎯 使用流程

### 创建任务并拆解

1. **创建主任务**
   ```
   点击"新建任务"
   → 填写任务信息
   → 选择模板（可选）
   → 创建
   ```

2. **拆解为子任务**
   ```
   打开任务详情
   → 点击"添加子任务"
   → 填写子任务信息
   → 选择负责人
   → 设置工时和截止时间
   ```

3. **查看进度**
   ```
   看板视图 → 整体进度
   任务详情 → 子任务进度条
   员工页面 → 个人工作量
   ```

4. **完成任务**
   ```
   点击子任务 → 自动切换状态
   → 系统自动更新所有关联数据
   → 进度条自动更新
   → 员工负载自动调整
   ```
